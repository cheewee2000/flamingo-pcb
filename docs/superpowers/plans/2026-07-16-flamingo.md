# Flamingo Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A prompt-first PCB CAD app: Claude drives board design via MCP, user manually arranges in a live browser PCB view, output is JLCPCB-ready gerbers + BOM + CPL using only LCSC parts.

**Architecture:** npm-workspaces TypeScript monorepo. Pure `engine` library (data model, ops, netlist, DRC, SVG render) + `parts` (LCSC/EasyEDA) + `fab` (gerber/BOM/CPL/DSN/SES) + `server` (document host, WS, MCP over HTTP, Freerouting runner) + `ui` (Vite canvas app). Server owns one live board doc; every mutation is a validated op broadcast over WebSocket.

**Tech Stack:** Node 22, TypeScript 5 strict, npm workspaces, Vitest, Vite (ui), `@modelcontextprotocol/sdk` (MCP streamable HTTP), `polygon-clipping` (boolean poly ops), `@resvg/resvg-js` (SVG→PNG), `archiver` (zip), Freerouting JAR (external, Java).

**Spec:** `docs/superpowers/specs/2026-07-16-flamingo-design.md` — read it first; it overrides nothing here but fills in intent.

## Global Constraints

- Units: **mm everywhere in the engine, y-up**. Angles degrees CCW. Convert only at boundaries (EasyEDA parse: 10 mil units → mm ×0.254, y-down → y-up; UI canvas y-down; gerber is y-up natively).
- Copper layers: 2, 4, or 6. Through vias only. Layer ids exactly: `F.Cu In1.Cu In2.Cu In3.Cu In4.Cu B.Cu F.Silk B.Silk F.Mask B.Mask F.Paste B.Paste Edge`.
- Board file: pretty-printed JSON `*.flamingo`, `formatVersion: 1`.
- Server port **4242**: UI at `/`, REST under `/api/`, WS at `/ws`, MCP at `/mcp`.
- All packages: `"type": "module"`, TS `strict: true`, ESM imports with `.js` extensions in emitted paths (use `moduleResolution: "bundler"` in ui, `NodeNext` elsewhere).
- Tests: Vitest, colocated in `packages/*/test/*.test.ts`. Run all: `npm test -ws --if-present`.
- Commit after every task (conventional commits). Never commit `node_modules`, `dist`, `~/.flamingo`.
- Network fixtures: parser tests use committed JSON fixtures, never live HTTP.
- Pin refs are `"<REFDES>.<PADNUMBER>"` strings, e.g. `"U1.3"`.
- Ids (`Track.id` etc.): `crypto.randomUUID()`.

```ts
// types.ts
export type LayerId = 'F.Cu'|'In1.Cu'|'In2.Cu'|'In3.Cu'|'In4.Cu'|'B.Cu'
  |'F.Silk'|'B.Silk'|'F.Mask'|'B.Mask'|'F.Paste'|'B.Paste'|'Edge';
export interface Point { x: number; y: number }
export type PathSeg =
  | { type:'line'; start: Point; end: Point }
  | { type:'arc'; start: Point; end: Point; center: Point; cw: boolean };
export interface Pad {
  number: string;
  shape: 'rect'|'oval'|'circle'|'polygon';
  at: Point;                 // relative to footprint origin
  rotation: number;          // deg CCW, relative to footprint
  size: { w: number; h: number };
  polygon?: Point[];         // when shape==='polygon', relative to `at`
  drill?: { diameter: number; slotLength?: number; plated: boolean };
  layer: 'top'|'bottom'|'through';
}
export interface SilkItem =
  | { kind:'line'; start:Point; end:Point; width:number }
  | { kind:'arc'; start:Point; end:Point; center:Point; cw:boolean; width:number }
  | { kind:'circle'; center:Point; radius:number; width:number }
  | { kind:'text'; at:Point; text:string; height:number; rotation:number };
export interface Footprint {
  name: string; lcsc: string;
  pads: Pad[]; silk: SilkItem[]; courtyard: Point[][];
}
export interface ComponentInst {
  refdes: string; lcsc: string; footprint: Footprint;
  at: Point; rotation: number; side: 'top'|'bottom';
  fields: { value?:string; description?:string; mfr?:string; package?:string; basic?:boolean };
}
export interface Net { name: string; class: string; pins: string[] }
export interface NetClass { name:string; trackWidth:number; clearance:number; viaDrill:number; viaDiameter:number }
export interface Track { id:string; layer:LayerId; width:number; net:string; seg:PathSeg }
export interface Via { id:string; at:Point; drill:number; diameter:number; net:string }
export interface Zone { id:string; layer:LayerId; net:string; polygon:Point[]; clearance:number; minWidth:number; thermal:{gap:number; spokeWidth:number}; fill?:Point[][] }
export interface Keepout { id:string; layers:LayerId[]|'all'; polygon:Point[]; keepout:{copper:boolean; via:boolean} }
export interface MountingHole { id:string; at:Point; drill:number; padDiameter:number; plated:boolean }
export interface SilkText { id:string; layer:'F.Silk'|'B.Silk'; at:Point; text:string; height:number; rotation:number }
export interface Board {
  formatVersion: 1; name: string; copperLayers: 2|4|6;
  outline: PathSeg[]; keepouts: Keepout[]; holes: MountingHole[];
  components: ComponentInst[]; nets: Net[]; netClasses: NetClass[];
  tracks: Track[]; vias: Via[]; zones: Zone[]; silk: SilkText[];
  rules: 'jlcpcb-2l'|'jlcpcb-4l'|'jlcpcb-6l';
}
// board.ts
export function newBoard(name: string, copperLayers: 2|4|6): Board; // default netClass 'default' {trackWidth:0.25, clearance:0.2, viaDrill:0.3, viaDiameter:0.6}, rules matching layer count
export function serializeBoard(b: Board): string;   // pretty JSON
export function parseBoard(json: string): Board;    // throws Error with message on invalid/unknown formatVersion
// layers.ts
export function copperLayersOf(b: Board): LayerId[];       // e.g. 4 → ['F.Cu','In1.Cu','In2.Cu','B.Cu']
export function isCopper(l: LayerId): boolean;
```

Note: 4-layer uses `In1.Cu,In2.Cu`; 6-layer `In1..In4`.

### Task 3: Engine — ops + undo

**Files:** Create `packages/engine/src/ops.ts`. Test `packages/engine/test/ops.test.ts`.

**Produces:**

```ts
export type Op =
  | { op:'placeComponent'; refdes:string; lcsc:string; footprint:Footprint; at:Point; rotation:number; side:'top'|'bottom'; fields:ComponentInst['fields'] }
  | { op:'moveComponent'; refdes:string; at?:Point; rotation?:number; side?:'top'|'bottom' }
  | { op:'removeComponent'; refdes:string }            // also strips its pins from nets, deletes its nets' tracks? NO — keeps tracks (DRC will flag); removes pins from nets
  | { op:'setOutline'; outline:PathSeg[] }
  | { op:'addKeepout'; keepout:Omit<Keepout,'id'> } | { op:'addZone'; zone:Omit<Zone,'id'|'fill'> }
  | { op:'addHole'; hole:Omit<MountingHole,'id'> } | { op:'addSilkText'; text:Omit<SilkText,'id'> }
  | { op:'removeItem'; id:string }                     // any id-bearing item (keepout/zone/hole/silk/track/via)
  | { op:'connectPins'; net:string; pins:string[] }    // creates net if absent (class 'default'); appends pins; a pin may belong to ONE net (reject if already in another)
  | { op:'disconnectPins'; pins:string[] }
  | { op:'renameNet'; from:string; to:string }
  | { op:'createNetClass'; netClass:NetClass } | { op:'assignNetClass'; net:string; class:string }
  | { op:'addTrack'; track:Omit<Track,'id'> } | { op:'addVia'; via:Omit<Via,'id'> }
  | { op:'addTracks'; tracks:Omit<Track,'id'>[]; vias:Omit<Via,'id'>[] }  // bulk (SES import)
  | { op:'unroute'; net?:string }                      // delete tracks+vias of net, or all
  | { op:'setBoardMeta'; name?:string; copperLayers?:2|4|6 };
export interface OpResult { ok: true; board: Board; createdIds: string[] } 
export interface OpError { ok: false; error: string }
export function applyOp(b: Board, op: Op): OpResult | OpError; // pure: returns NEW board (structuredClone), never mutates input
```

Validation (reject with actionable `error`): duplicate refdes on place; unknown refdes/net/id; pin format invalid or pad number not on that footprint; pin already in another net; layer not valid for stackup (e.g. `In3.Cu` on 4-layer); copperLayers change that orphans tracks on now-invalid layers.

Undo is the server's job (Task 7) via snapshot stack — `applyOp` purity makes that trivial.

Parsing the EasyEDA API (`GET https://easyeda.com/api/products/{LCSC}/components?version=6.4.19.5`, JSON at `result.packageDetail.dataStr`):
- `dataStr.head.x/y` is the canvas origin; all coords are relative to it after subtraction. **Units: 1 unit = 10 mil = 0.254 mm.** EasyEDA canvas is y-down: negate Y.
- `dataStr.shape` is an array of `~`-delimited strings. Handle: `PAD~<shape>~<x>~<y>~<w>~<h>~<layerId>~<net>~<number>~<holeRadius>~<points>~<rotation>~...` (shape ELLIPSE/RECT/OVAL/POLYGON; layerId 1=top,2=bottom,11=multi/through; holeRadius>0 → drill=2×radius; slot via holeLength field), `TRACK~<w>~<layer>~...~<pts>` (layer 3=F.Silk,4=B.Silk) → silk lines, `ARC`, `CIRCLE` → silk, `SOLIDREGION`/`RECT` on layer 99/courtyard → courtyard.
- The parser MUST log-and-skip unknown shape types, never crash.
- `result.dataStr.head.c_para` and top-level `result` carry mpn/package/`SMT` info; Basic/Extended: `result.SMT === true && result.basic`… **verify empirically**: fetch a known Basic part (C25804, 10k 0603) and a known Extended part, inspect JSON, adapt, and record the actual field paths in code comments.
- Search: primary endpoint `POST https://easyeda.com/api/components/search` (form fields `type=3`, `doctype[]=2`, `wd=<query>`, `pageSize`); if that fails in practice, fall back to `GET https://wmsc.lcsc.com/ftps/wm/search/global-search?keyword=<q>`. Verify with curl during the task; keep whichever works, put the OTHER behind a fallback. Search results must populate `basic` when derivable.
- Fixtures: run a small script once to download raw API JSON for C25804 (0603 R), C2040 (SOT-23? verify), a QFN, an ESP32-S3 module (C2913204 or similar — verify exists), a USB-C receptacle (C165948), commit as fixtures. Parser tests run offline against fixtures and assert pad counts, a specific pad's mm position (hand-converted), drill presence on THT, courtyard non-empty.

### Task 11: Fab — Specctra DSN export / SES import + Freerouting runner

**Files:** Create `packages/fab/src/dsn.ts`, `src/ses.ts`; `packages/server/src/route.ts`; register MCP tools `autoroute{nets?, passes?}`, `unroute{net?}` in `mcp.ts`. Tests `packages/fab/test/dsn.test.ts`, `test/ses.test.ts` (fixture SES committed).

**Produces:**

```ts
export function exportDSN(b: Board): string;
export function importSES(ses: string, b: Board): { tracks: Omit<Track,'id'>[]; vias: Omit<Via,'id'>[] }; // resolution-scaled back to mm, net names mapped
```

DSN essentials: `(pcb (parser (host_cad flamingo)) (resolution um 10) (structure (layer <name> (type signal))… (boundary (path pcb 0 <outline pts>)) (keepout …) (rule (width <default µm>) (clearance <µm>))) (placement (component <fp-image-id> (place <refdes> x y <side front|back> <rot>)…)) (library (image <fp-image-id> (outline…)(pin <padstack> <padnum> x y))… (padstack …)) (network (net <name> (pins R1-1 U1-3…)) (class <netclass> <nets…> (rule (width…)(clearance…))))`. Coordinates µm integers, y-up. Pin refs `REFDES-PADNUM`. One padstack per distinct pad geometry (dedupe by hash).
